## What This Repo Does

This repository implements the **OS-APOW** (Orchestration Service — Autonomous Pipeline for Orchestrated Workflows) system. It replaces interactive AI coding with persistent, event-driven infrastructure where:

1. GitHub Issues serve as "execution orders"
2. A Sentinel service polls for queued tasks and dispatches them
3. An opencode Worker (in a DevContainer) autonomously generates code, runs tests, and submits PRs
4. GitHub Labels act as a state machine tracking task lifecycle

## Architecture (4 Pillars)

- **The Ear** — FastAPI webhook receiver for sub-second event ingestion (Phase 2)
- **The State** — GitHub Issues/Labels/Milestones as transparent, auditable work queue
- **The Brain** — Python async Sentinel that polls, claims, and dispatches tasks
- **The Hands** — Isolated DevContainer running opencode CLI with LLM-driven agents

## How to Run

```bash
# Validation (lint + scan + test)
pwsh -NoProfile -File ./scripts/validate.ps1 -All

# Start devcontainer opencode server
./scripts/start-opencode-server.sh

# Run orchestrator prompt
./scripts/devcontainer-opencode.sh prompt -f <prompt-file>
```
